Critical resolution of D-27 (CONTEXT investigation flag) — settled at plan time using local `governor 0.6.3` source: `governor::clock::DefaultClock` resolves to `QuantaClock` when both `std` and `quanta` features are enabled (default). `QuantaClock::now()` reads `quanta::Clock::raw()` which uses the OS monotonic clock and IS NOT honored by `tokio::time::pause()`/`tokio::time::advance()`. Therefore the `retain_recent` test (D-25 #4) constructs the limiter explicitly with `governor::clock::FakeRelativeClock` and uses `clock.advance(Duration::from_secs(120))`. Boundary 429 tests (D-24 #4 and #5) DO NOT need time control — they exhaust the burst (10/30) in a tight loop. No real `tokio::time::sleep` anywhere in the suite (avoids CI flakes).

Critical resolution of the per-pubkey-fires-first oracle (checker blocker) — D-01 (per-pubkey burst 10) is STRICTLY less than D-02 (per-IP burst 30). Any test that fixes a single `trade_pubkey` and varies only IPs will exhaust the per-pubkey limiter at iteration 11, NEVER reaching the per-IP path. The fix is structural: every test whose intent is to exercise the per-IP limiter ROTATES `trade_pubkey` across iterations (32+ distinct 64-hex values like `format!("{:0>64x}", i)`). Every test whose intent is to exercise the per-pubkey limiter ROTATES IPs while fixing the pubkey. Both halves of the byte-equality test follow this rule independently. Each per-IP test additionally asserts the iteration index where the first 429 appears matches the expected boundary (`>= IP_BURST + 1` = 31), proving structurally that the captured 429 came from the per-IP path and not the per-pubkey path.

/// Stub PushService recording every dispatch call (D-23).
///
/// `calls`: Arc<Mutex<Vec<(device_token, platform)>>>. Each successful or
/// attempted send is appended; tests assert `calls.lock().await.len()` and

/// `supports`: which platforms `supports_platform` returns true for. Lets a
/// test exercise the `NoBackendForPlatform` dispatch path.
/// `fail`: when true, `send_to_token` returns Err("stub forced failure").
pub struct StubPushService {
    pub calls: Arc<Mutex<Vec<(String, Platform)>>>,
    pub supports: Vec<Platform>,
    pub fail: bool,
}

impl StubPushService {
    pub fn new(supports: Vec<Platform>) -> Self {
        Self {
            calls: Arc::new(Mutex::new(Vec::new())),
            supports,
            fail: false,
        }
    }
}

#[async_trait]
impl PushService for StubPushService {
    async fn send_to_token(
        &self,
        device_token: &str,
        platform: &Platform,
    ) -> Result<(), Box<dyn std::error::Error + Send + Sync>> {
        self.calls
            .lock()
            .await
            .push((device_token.to_string(), platform.clone()));
        if self.fail {
            Err("stub forced failure".into())
        } else {
            Ok(())
        }
    }

    fn supports_platform(&self, platform: &Platform) -> bool {
        self.supports.contains(platform)
    }

/// Test-time per-pubkey quota: 30/min burst PUBKEY_BURST. Same as production
/// per D-01 / Plan 01 — the boundary test exhausts the burst (10) in a
/// tight loop and expects 429 on call 11+.
pub fn test_per_pubkey_quota() -> Quota {
    Quota::per_minute(NonZeroU32::new(30).unwrap())

/// Build a test AppState + per-IP limiter pair using fresh limiters.
/// Returns (state, per_ip_limiter, stub) so callers can assert against the
/// stub recordings after exercising the app.
pub fn make_app_state(
    stub: Arc<StubPushService>,
) -> (AppState, Arc<PerIpLimiter>) {
    let services: Vec<(Arc<dyn PushService>, &'static str)> =
        vec![(stub.clone() as Arc<dyn PushService>, "stub")];
    let dispatcher = Arc::new(PushDispatcher::new(services));

    let mut salt_bytes = [0u8; 32];
    rand::thread_rng().fill_bytes(&mut salt_bytes);
    let notify_log_salt: Arc<[u8; 32]> = Arc::new(salt_bytes);

    let token_store = Arc::new(TokenStore::new(48, notify_log_salt.clone()));
    let semaphore = Arc::new(Semaphore::new(50));

    let per_pubkey_limiter: Arc<PerPubkeyLimiter> =
        Arc::new(RateLimiter::keyed(test_per_pubkey_quota()));
    let per_ip_limiter: Arc<PerIpLimiter> =
        Arc::new(RateLimiter::keyed(test_per_ip_quota()));

    let state = AppState {
        token_store,
        dispatcher,
        semaphore,
        notify_log_salt,
        per_pubkey_limiter,
    };

    (state, per_ip_limiter)
}

/// Initialise an in-process actix-web service against the production

        Error = actix_web::Error,
