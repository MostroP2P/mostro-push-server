## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| mobile client -> POST /api/notify | unauthenticated; the only field accepted is `trade_pubkey`. Source IP is observable to the runtime but MUST NOT be logged or persisted. |
| mobile client -> X-Request-Id header | inbound header is untrusted; MUST be stripped before reaching the handler so the client cannot correlate its own requests with server state. |
| handler -> tokio::spawn -> dispatcher.dispatch_silent -> FCM/UnifiedPush | spawn is detached from response; permit bounds the in-flight count to 50; permit drops at task end. |
| handler -> log lines | every pubkey identifier in `info!`/`warn!` lines from `/api/notify` paths comes from `log_pubkey(salt, pk)` and never the raw `trade_pubkey` or its hex prefix. |
